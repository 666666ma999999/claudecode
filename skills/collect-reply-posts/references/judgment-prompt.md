# Stage 3 判定 rubric（採否＋論点＋見出し案）

Stage 2 で拾った X 投稿群（URL ＋ 本文 ＋ 分かれば reply 概数）に対し、**1回のLLM処理**で以下を行う。記事ネタ素材としての筋の良さを判定するのが目的。「reply が多い」は前提条件であって採用理由ではない。

## 判定軸（各投稿）

| 軸 | 見るもの | 捨てる条件 |
|---|---|---|
| **議論密度** | reply が賛否で割れている / 専門的な掘り下げ / 有益な追加情報が連なっている | 挨拶・絡み・「フォロバして」・スパム・bot だけの reply |
| **トピック適合** | AI / Claude Code / 開発・AI活用の実務に関わる | 政治・芸能・無関係なバズ |
| **記事化余地** | そこから書ける角度がある（賛否・誤解・実体験・比較・反論） | 単発の事実報告で広げようがない |

- `discussion_score` 0-5（議論密度）。**3未満は却下**。
- `verdict`: 採用 / 保留 / 却下。

## 出力フォーマット（採用分のみ・x-stock に渡せる形）

各採用投稿について、以下を出す。Stage 4 でこのまま `/x-stock` に渡す。

```
### 採用 [discussion_score: N]
- title（見出し案・強い順に2本）:
    1. <見出しA>
    2. <見出しB>
- 論点（何で議論になっているか・1行）: <...>
- 角度（記事の切り口）: <...>
- 根拠 reply（議論を象徴する反応の要約・1-2点）: <...>
- 出典: <投稿URL>
- tags: [x-reply, <論点タグ>]
```

却下・保留は1行ずつ理由だけ（`却下: <URL> — 挨拶replyのみ` の形）でよい。

## 注意

- **見出しは煽りでなく「議論の核」を突く**。reply が割れた理由＝読者の関心なので、そこを見出しに。
- 同じ論点の投稿が複数あれば、1つの「論点」にまとめて代表投稿を出典にしてよい（単発バズより"うねり"を優先）。
- 既存 `x-article-stock.md` と明らかに重複する角度は「既出かも」と添えるだけ（排除は人が判断）。
- モデル: 通常セッション/安いモデルで十分。Fable5 を使うなら候補10件以下に絞った後の本処理を1回だけ。
