# Opponent Review スキル

## 概要

2つの対立する視点を持つSubAgentに意図的に議論させることで、
単一視点では見逃しがちなリスク・問題点・代替案を発見する。

security-twin-audit のセキュリティ特化版に対し、こちらは汎用的な意思決定・レビュー用。

## ペア定義（用途別）

| 用途 | Agent A（推進役） | Agent B（反証役） | 発動条件 |
|------|-------------------|-------------------|----------|
| **Architecture** | Proposer: アプローチの利点を主張 | Skeptic: リスク・将来負債・代替案を指摘 | アーキテクチャ判断、技術選定 |
| **Refactor** | Simplifier: コード簡素化・DRY推進 | Regression Hunter: 機能喪失・エッジケース退行を検知 | 大規模リファクタリング |
| **Performance** | Optimizer: 速度・効率の最適化提案 | Correctness Guard: 最適化による正確性低下を指摘 | 性能改善、キャッシュ導入 |
| **Requirements** | Builder: 実装可能性と価値を主張 | Scope Challenger: スコープ肥大・YAGNI違反を指摘 | 要件レビュー、仕様決定 |
| **Postmortem** | Root Cause: 根本原因の仮説を主張 | Alternative Cause: 別の原因仮説を提示 | 障害分析、バグ調査 |

## 実行手順

### Step 1: ペア選択

対象のタスク・判断内容から適切なペアを選択する。
ユーザーが明示しない場合は、コンテキストから自動判定する。

### Step 2: 対象情報の収集

両Agentに渡す共通情報を収集:
- 対象コード / 設計ドキュメント / PRの差分
- 現在の制約条件（技術スタック、パフォーマンス要件等）
- 判断が必要な論点

### Step 3: 並列SubAgent起動

2つのSubAgentを**並列**で起動する。各Agentには以下を渡す:

```
Agent A（推進役）プロンプト:
「あなたは[ペアのAgent A名]です。以下の[対象]について、
[推進する視点]から分析してください。
利点・実現可能性・価値を重点的に論じてください。
ただし、弱点も認識している場合は正直に述べてください。

対象: [共通情報]
論点: [判断が必要な内容]」

Agent B（反証役）プロンプト:
「あなたは[ペアのAgent B名]です。以下の[対象]について、
[反証する視点]から分析してください。
リスク・見落とし・代替案を重点的に論じてください。
ただし、良い点も認識している場合は正直に述べてください。

対象: [共通情報]
論点: [判断が必要な内容]」
```

### Step 4: 統合判断

両Agentの結果を受けて、メインAgentが統合レポートを作成:

```markdown
## Opponent Review Report

### 論点
[判断が必要だった内容]

### Agent A（推進役）の主張
- [要点1]
- [要点2]

### Agent B（反証役）の主張
- [要点1]
- [要点2]

### 合意点
- [両者が一致した点]

### 対立点
| 論点 | Agent A | Agent B |
|------|---------|---------|
| ... | ... | ... |

### 統合判断
[両者の分析を踏まえた最終判断と理由]

### 残存リスク
[判断後も残るリスクとその緩和策]
```

### Step 5: ユーザー提示

統合レポートをユーザーに提示し、判断を仰ぐ。
ユーザーが追加の論点を指摘した場合、Step 3に戻って再議論。

## 簡易版（SubAgent 1つで実施）

コスト・時間の制約がある場合、1つのSubAgentに「Devil's Advocate」ロールを割り当て:

```
「以下の判断について、意図的に反対の立場から分析してください。
この判断の弱点・リスク・見落としを指摘し、代替案があれば提示してください。
判断: [内容]」
```

## 発動基準

以下に該当する場合、このスキルの使用を提案する:
- 影響範囲が3ファイル以上のアーキテクチャ判断
- 「AかBか」の二択で迷っている設計判断
- 大規模リファクタリングの方針決定
- 性能改善で正確性とのトレードオフがある場合
- 障害の根本原因が複数の仮説で説明可能な場合
