# precedent-check — 前例検索してから作る

新規成果物の**形式・置き場所・スタイルを決める前**に必ず実行する。目的は「MD で作ったら実は Web ページ版の前例があった」類の手戻り防止。

## 手順（上から順・見つかった時点で打ち切り可）

1. **claude-mem 検索（利用可能時のみ・任意）**: `mcp__plugin_claude-mem_mcp-search__search` がツール一覧に存在する場合のみ実行（plugin 停止中は不在＝スキップし残り 3 経路で足りる。2026-07-13 軽量化で停止）。検索語例: "viewer html", "ダッシュボード", "キーワード集"
2. **ファイルシステム検索**: `find ~/Desktop/biz ~/Desktop/prm -maxdepth 3 -name "*<種類>*.<拡張子>" -not -path "*/node_modules/*"`（html/py/md それぞれ。viewer/dashboard/report/viewer 系の命名揺れも試す）
3. **ルーティング逆引き**: `grep -i <キーワード> ~/.claude/rules/30-routing.md`（既存スキル/コマンドが同じことをしていないか）
4. **vault 検索**: `02_Ai/` 配下の `_ope.md`（各プロジェクト MOC）の「repo 側正本」表を grep

## 見つかったら

- 場所・形式・生成元スクリプトを**ユーザーに報告してから**形式を決める（前例に揃えるのが既定。乖離するなら理由を 1 文で確認）
- スタイル資産（CSS・テンプレ・慣例）は流用する

## 見つからなくても

「前例なし（検索済み: FS / routing / vault MOC ＋ mem-search は利用可能時のみ）」と 1 行報告してから新規作成に進む（無言で新規作成しない）。
