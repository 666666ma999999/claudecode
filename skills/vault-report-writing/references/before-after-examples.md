# Before → After 例（悪い例 → 改善例）

短い対比で「何を直すと読まれるか」を示す。

## 1. 結論が埋もれている → BLUF

**Before**（結論が最後）
```markdown
## 背景
... 長い前提 ...
## 分析
... 表が続く ...
## まとめ
よって A 案を推奨します。
```
**After**（結論先出し）
```markdown
> [!success] 結論
> **A 案を推奨**。理由は LTV +17% / CPA 据え置き / 学習リセットなし。

> [!abstract]- 背景・分析（開く）
> ...
```

## 2. 全部太字・多色 → 1 色強調

**Before**: `**重要**な**指標**が**悪化**、**至急****対応**を…`（全部強調＝何も強調されない）
**After**: 重要指標が悪化。==CPA +36%== が閾値超え。→ `[!danger]` に 1 つだけ。

## 3. ヘッダー欠落の壊れた表 → 列定義を復活

**Before**（列の意味不明・実際に v2 で発生）
```markdown
| 占いプライム-Max | 自動 | 維持 | ¥1,182k | 1,456 | ¥811 |
| ---------------- | ---- | ---- | ------- | ----- | ---- |
```
**After**（ヘッダー行を足す）
```markdown
| 広告(CP) | 種別 | 強化 | 現状cost/月 | 現状CV | 現状CPA |
| --- | --- | --- | --: | --: | --: |
| 占いプライム-Max | 自動 | 維持 | ¥1,182k | 1,456 | ¥811 |
```
→ **装飾より先に「壊れた構造」を直す**。

## 4. 手書き索引表（陳腐化） → 動的索引

**Before**: MOC に施策一覧を手書き → 更新漏れで drift。
**After**:
```markdown
​```dataview
TABLE status, last_updated FROM #measure SORT last_updated DESC
​```
```
→ 各ノートのメタを直せば索引が自動追従。

## 5. 補足が本文を埋める → foldable に隔離

**Before**: 各商品の選定理由が本文に長々 → スキャンできない。
**After**: 要点は表、詳細は `> [!note]- 補足` に畳む → 上から読めばスキャン / 開けば網羅。

## 6. 二重管理 → embed

**Before**: 司令塔と各レポートに同じ要約をコピペ → どちらが正本か不明・drift。
**After**: `![[施策A#結論]]` で正本を 1 箇所に → 参照で一元化。

## 7. 成功参照例（2026-06-24 広告bpoチャンネル分析・司令塔ボード清書）

本 skill の規約（🎯ゴール先出し → BLUF → 逆ピラミッド）に準拠して清書し、codex 敵対レビューで見つかった低信頼推論は本文に書かず Q1〜Q9 の確認質問に構造化 → ユーザー確認後に次工程（ファクトチェック）へ。**低信頼の推論は断定で書かず「確認質問」ブロックに隔離する**のが通ったパターン。
