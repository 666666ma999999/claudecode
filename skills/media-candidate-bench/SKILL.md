# メディア候補抽出ベンチ

「動画から良いコマ/区間を自動で出す」機能の方式選定・回帰確認の手順。感覚で選ぶと**先頭偏重・別人混入・ブレコマ採用**が起きる（出典: reading-factory 種コマ抽出 2026-07-14 実測）。

## ベンチ手順

### 0. fixture を固定
- [ ] 評価用動画1〜2本と**正解ラベル**（対象人物が単独で映る時間帯・使ってよい/だめの例）を先に固定。以後の方式比較は全て同じ fixture で行う

### 1. 抽出パイプラインの基本形
- [ ] フレームサンプリング間隔を明示（例: 1fps でスキャン→当たり区間だけ 5fps）
- [ ] 鮮明度スコア＝顔が来る領域の**エッジ分散（Laplacian分散）**をベースライン に（実装3行・無償）
- [ ] シーンカット検出だけでは**カット内の複数良フレーム**を取れないことに注意（区間ベースと フレームベースは別物）

### 2. 単独人物判定の方式比較（コスト×精度）
| 方式 | 測ること |
|---|---|
| 顔検出ライブラリ | 再現率・誤採用率・依存追加コスト（Dockerイメージ肥大） |
| Vision LLM | 単価×フレーム数（**全フレーム投げは費用暴走**・既存設計で不採用の前例確認）・レート制限 |
| 人手（賢い候補提示のみ自動） | 候補数を絞れているか・見落とし率 |
- [ ] 判定は**同じ fixture・同じ評価表**で比較し、採用方式と根拠を1枚に記録

### 3. 候補提示の質
- [ ] **top-K の時系列偏重**を点検（時間順で先頭から切ると後半の最良フレームに到達しない → 全スキャン後にスコア順 top-K）
- [ ] 同一ショットの**近接重複を排除**（隣接フレームはほぼ同一 — クラスタリングか最小時間間隔）
- [ ] 連続コマを出すときは「同じ場面の保険2枚」だと**明示**（利用者が全部別candidateと誤解する・実例あり）
- [ ] 候補の来歴（元動画・時刻・抽出設定）を候補ごとに保存

### 4. 配信・権利
- [ ] 人物候補画像は**認証付きエンドポイント**で配信（無認証 static に置かない）
- [ ] 素材の許諾状態を候補メタデータに持たせ、未許諾は本番昇格をブロック
