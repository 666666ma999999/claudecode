# 要件定義ワークシート & 要約テキストフォーマット

出典: デジタル庁「要件定義ワークシート」（worksheet.pptx）+ 「要約テキストフォーマット」（format.docx）

## 要件定義ワークシート

ダッシュボード/グラフ作成前に整理すべき項目。ユーザーからの依頼を受けた時に、以下を確認する。

### 1. プロジェクトの基本情報

| 項目 | 確認すること | 記載例 |
|---|---|---|
| 達成すべき目標 | プロジェクトが成功と言える状態は何か？ | 売上目標未達時に問題点を発見し打開策を検討できる |
| 関係者 | 意思決定者・ダッシュボードを見る人は誰か？ | プロダクトオーナー、マーケ担当、経営層 |
| ダッシュボードの制約 | 掲載媒体や画面的制約は何か？ | タブレット対応必要、3枚以内でまとめる |
| データの制約 | データ項目や更新頻度、分解粒度の制約は？ | 週次更新、顧客データのみ月次、速報値注意 |
| リスクと許容度 | 考慮すべきことは何か？ | ラベル変更の可能性、キャンセル反映の遅延 |
| スケジュール | いつまでに必要か？ | — |

### 2. ダッシュボードの目的（5W1H）

| 問い | 確認すること |
|---|---|
| **Why**（目的） | ダッシュボードの目的は何か？ |
| **What, so What**（情報と行動） | どんな意思決定や行動のために、どのような情報を知るべきか？ |
| **Who, When, Where**（利用シーン） | 誰が、いつ、どこで見るのか？ |
| **How**（機能・データ） | 求められる機能やデータ項目は何か？必要な更新頻度は？ |

### 3. 必要とされるデータ（記載例）

- 売上データ: 週次での実績値と目標値の推移、部門別、顧客属性別、対応店舗別
- 顧客データ: 週次での推移
- 対応店舗データ

### 4. 必要とされる機能（記載例）

- 売上や顧客数が伸びている優秀な対応店舗が見つけられる
- 今後、開拓すべきエリアや店舗が見つけられる
- 売上が目標通りの推移をしているか確認して、メンバーや上層部に報告できる

## グラフ生成時の簡易質問テンプレート

ユーザーの依頼から以下を抽出する。不足があれば確認する。

```
1. 誰に見せるか？ → [経営層 / チームメンバー / 外部 / 自分]
2. 何を判断してほしいか？ → [具体的な判断・行動]
3. 何と比較するか？ → [前年比 / 目標値 / 他部門 / なし]
4. どの期間を見るか？ → [月次 / 週次 / 四半期]
5. データの制約は？ → [更新頻度 / 速報値 / 欠損]
```

## 要約テキストフォーマット

ダッシュボードやグラフの内容を、テキストで要約する際の標準フォーマット。

### 構成

```markdown
# [ダッシュボードのタイトル]

[データ更新日] 時点の数値

## [セクション1のタイトル]

[主な数字やハイライトを文章で記述]

## [セクション2のタイトル]

[同上]

## データの定義について

[データの定義についての説明、またはリンク]
```

### 記載例

```markdown
# マイナンバーカードの普及状況

2024年1月21日 時点の数値

## マイナンバーカード

マイナンバーカードの累計申請件数（有効件数）は99,247,212件です。
累計交付枚数は97,645,168件で、人口に対する申請件数率は79.1%です。
1日あたり（直近4週平均）の申請件数は9,308件です。
先週の合計は82,573件で、前週の合計の86,121件と比べて3,548件減っています。

## データの定義について

本内容におけるデータおよび数値の内訳については、
[データの定義ページ]をご確認ください。
```

### グラフ生成時の活用

- PNG出力時に、同じ内容のテキスト要約も生成すると丁寧
- 主要数値 + 前期比 + 傾向を1段落で書く
- 「〇〇は△△件で、前月比□□%増加しています」の形式
