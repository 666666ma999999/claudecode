# dashboard-design-guide

## このスキルの役割

このスキルは、派手さやSNS映えではなく、意思決定に必要な情報を最短で正確に伝えるための可視化を行う。

最優先は次の2点。

1. 知りたいことを知れる
2. 誤解を生まない

入力は `CSVデータ` と `何を伝えたいか` の短い指示。
出力は `matplotlib` による `PNG` グラフ、または複数グラフを並べた静的ダッシュボード画像。

## 発動条件

以下の依頼ではこのスキルを使う。

- 経営会議、役員会、定例報告、行政説明資料向けのグラフ作成
- KPIダッシュボード、業績モニタリング、施策進捗の可視化
- CSVから、見やすく、誤解の少ないグラフ画像を作る依頼
- 「何を伝えるべきか」から逆算してチャート型を選ぶ必要がある依頼
- 全体指標と詳細指標をセットで示す可視化
- 比較対象、前年比、目標値、基準値を含めた可視化

以下では原則として使わない。

- X投稿用の強い演出、金融相場向けの煽り表現
- ダークテーマ前提のインパクト重視グラフ
- 金融時系列に特化したシグナル検出

## 既存 `data-visualization` スキルとの関係

別スキルとして分離する。

- `data-visualization` は金融データとX投稿向けの「インパクト最適化」が中心
- 本スキルは経営資料とダッシュボード向けの「判断支援」と「誤解防止」が中心

使い分け:
- `data-visualization`: 金融、相場、X投稿、強い訴求
- `dashboard-design-guide`: 経営資料、行政資料、KPI監視、役員向け説明

## 実行フロー

毎回この順番で進める。

1. 伝えたいことを1文で定義する
2. 利用者が判断したいことを特定する
3. 主要指標と補助指標を分ける
4. 比較対象を決める
5. データの性質からチャート型を選ぶ
6. 色数を絞り、装飾を削る
7. タイトル、単位、定義、更新日、出典を明記する
8. PNGを書き出す前にチェックリストで検査する

## 入力解釈ルール

まずユーザーの指示から次を抽出する。

- 主メッセージ
- 想定読者
- 判断・行動
- 比較対象
- 時間軸の有無
- カテゴリ比較か、時系列か、構成比か
- 単位
- 集計粒度
- 更新日と出典の有無

主メッセージが曖昧なら、次の形式に圧縮してから設計する。

`誰に / 何を判断してほしいか / 何と比較して / どの期間を見るか`

## チャート選択ルール

チャートは見栄えで選ばず、問いに対して選ぶ。

### 基本決定木

- 時間変化・傾向を見せたい: `折れ線グラフ`
- 数量を比較したい: `棒グラフ`
- 構成比の時間変化を見せたい: `積み上げ面グラフ` または `積み上げ棒グラフ`
- ある時点の構成比を見せたい: `棒グラフ` を優先し、全体が明確でカテゴリ数が少ないときのみ `円グラフ`
- 達成率や実績対目標を見せたい: `横棒グラフ` または `箇条書きKPI + 基準線付き棒`
- 地域や部門など離散カテゴリの順位を見せたい: `棒グラフ`
- 増減寄与を見せたい: `ウォーターフォール`
- 2変数の関係を見せたい: `散布図`
- マトリクス状の強弱を見せたい: `ヒートマップ`（ただし主指標1つのみ）
- 全体指標と内訳を両方見せたい: `KPIカード + 詳細グラフ`

### 優先ルール

- 正確な比較が必要なら `円グラフ` より `棒グラフ`
- カテゴリが多いときは縦棒より `横棒`
- 系列が多すぎる折れ線は避ける
- 積み上げは総量と内訳の両方を読みたいときだけ使う
- 指標を1セルに詰め込まない
- 単位が異なる指標を1グラフに混在させない

### 推奨上限

- 円グラフのカテゴリ数: `5` 以下
- 1グラフの系列数: `5` 以下
- 1グラフの主色数: `1〜5`
- ラベルを読ませるカテゴリ数: `15` 以下
- ダッシュボード1画面の主メッセージ数: `1〜3`

## デザイン原則

### 1. 知りたいことを知れる

- 不要な情報を削る
- 並び順に意味を持たせる
- 重要箇所にだけ強弱を付ける
- 読解に時間をかけさせない
- 左上から右下へ、全体から部分へ配置する
- 利用者が次に取りたい行動につながる比較対象を置く

### 2. 誤解を生まない

- 前提知識がなくても読めるラベルにする
- 指標名だけでなく、何の値かを定義する
- 軸や尺度を恣意的に歪めない
- 出典、更新日、注記を記載する
- 単位、集計期間、対象範囲を明示する
- 色だけに分類を依存しない

### 3. ダッシュボード構造

- 左上に最重要KPI
- その右または下にトレンド
- 下段に内訳、要因、地域別、部門別などの詳細
- フィルターは上部か左部
- 画面全体は 16:9 を基本
- 6分割グリッドを基準に整列させる

## Do

- 全体指標と詳細グラフを併置する
- 項目の並び順に意味を持たせる
- 不要な要素を削除する
- 色数を `1〜5` 色に絞る
- タイトルに内容とデータ種別を入れる
- 凡例はグラフの近くに置く
- 棒グラフの原点は原則 `0`
- 目標値、前年比、前月比などの比較対象を置く
- 更新日と出典を明記する

## Don't

- 3D、ドロップシャドウ、過剰なグラデーション
- 装飾目的のアイコン多用
- 色だけで分類を識別
- 1セルに異なる単位の指標を同居させる
- 棒グラフの切れた軸で差を誇張する
- 項目順をランダムに並べる
- 凡例を離れた場所に置く
- 多数系列の折れ線を1枚に詰め込む
- 円グラフを多カテゴリで使う
- タイトルだけで結論が分からない表現

## タイトル・ラベル・メタ情報規約

### タイトル

タイトルは結論志向で書く。最低限、次を含める。

- 対象
- 指標
- 期間
- 単位またはデータ種別

良い例:
- `月次売上高の推移（2025年度、単位: 百万円）`
- `部門別営業利益の比較（2026年3月実績、単位: 百万円）`

弱い例:
- `売上`
- `実績`
- `グラフ1`

### フッター

各グラフまたは画面下部に記載する。

- `出典`
- `更新日`
- `注記`

例:
`出典: 基幹システム集計 / 更新日: 2026-04-06 / 注: 返品調整後`

## カラーパレット

デジタル庁公式カラーコード（7テーマ全色）は `references/color-palettes.md` に収録。
Python辞書・matplotlib用スニペットもそちらに記載。

通常はテーマを1つ選び、補助色は同一テーマ内で完結させる。複数テーマ混在は避ける。

出典:
- https://www.digital.go.jp/resources/dashboard-guidebook/color-palette/color-code
- https://github.com/digital-go-jp/policy-dashboard-assets

### 色の使い方

- 基本は `1テーマ + グレー`
- 最重要系列だけ強色（900）、その他は薄色（200/50）かグレー
- 警告は `red`（Error: #CE0000 or #850000）、良好は `green`（Success: #197A4B）
- コントラスト比は背景に対して最低 `3:1` を確保
- 色覚多様性を考え、線種、マーカー、ラベルでも区別する
- テキストは `#000000`（黒）、ラベルは `#626264`、背景は `#F8F8FB`

## レイアウトテンプレート

### テンプレートA: KPI + トレンド + 内訳

構成:
- 左上: 主要KPIカード 1〜2件
- 右上: 主指標の時系列折れ線
- 左下: カテゴリ別棒グラフ
- 右下: 注記、定義、更新日

### テンプレートB: 全体像 + 要因分解

構成:
- 左上: 総量KPI
- 右上: 目標線付き推移
- 下段左: 要因別棒グラフ
- 下段右: ウォーターフォール

### テンプレートC: 構成比 + 推移

構成:
- 左上: 合計KPI
- 右上: 現時点構成比
- 下段全幅: 構成比の時系列推移

## matplotlib実装ルール

- フォント: macOSは `Hiragino Sans`、Linuxは `Noto Sans CJK JP`
- `axes.unicode_minus = False`
- 保存: `PNG`, `dpi=150`
- 背景色: 白系 `#F8F8FB`
- 不要な枠線は消す
- 値ラベルは必要な箇所だけ
- 注釈は最大 `3` 個まで
- `figsize=(16, 9)` を基本
- `GridSpec(6, 6)` で配置

## 品質チェックリスト

出力前に必ず確認する。公式27項目の全文は `references/checklist.md` に収録。
以下はグラフ生成時に特に重要な項目を抽出したもの。

- [ ] 利用者が判断や行動に必要な情報が提供できているか
- [ ] 本当に必要な情報だけに絞り込めているか
- [ ] グラフ型は問いに対して妥当か
- [ ] 利用者が見たい数値へ優先的に目が向くか
- [ ] チャートや表の配列順序が認知しやすいか
- [ ] チャートと凡例が隣接し対応づけがわかりやすいか
- [ ] 色数が `1〜5` 色に収まっているか
- [ ] チャート要素（グリッドや枠線）を最小限にしているか
- [ ] 不要な装飾・3D・シャドウを避けているか
- [ ] タイトルや系列名が誤解のない表記か
- [ ] 棒グラフの原点は `0` か
- [ ] 更新日、出典、注記があるか
- [ ] 比較対象（目標値、前年比等）が掲載されているか
- [ ] 1セルに複数の異なる単位を詰め込んでいないか
- [ ] 色だけでチャート情報を識別していないか
- [ ] PNG化後に縮小表示しても読めるか

## アンチパターン

見つけたら修正する。

- 3D棒グラフ / ドロップシャドウ / 過度なグラデーション
- 2軸グラフで異単位を混在
- ランダム順のカテゴリ棒
- 円グラフの多カテゴリ化
- 注釈だらけで読解負荷が高い
- 主要メッセージが複数あって焦点がぼける
- 見た目優先で定義、単位、出典が欠ける

## 迷ったときの優先順位

1. 誤解を生まない
2. 判断に必要な比較ができる
3. シンプルで速く読める
4. 全体から詳細へたどれる
5. 見た目の統一感がある
