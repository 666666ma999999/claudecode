# vault-facts-ledger — 自社ファクト＆ローカルルール台帳の方法論

> 出自: prime_ad `AIads-facts.md`（2026-07-14 参照実装）。同日 Fable5×Codex 議論で全論点収束
> （1枚統合+意味分離 / key_findings と併存+振り分けテスト / 命名は `<sub>-facts.md` 標準）。

## 役割（1行）

**「分析や判断を始める前の前提」を1箇所で引ける台帳**。数字の真偽・定義・正本の所在・自社ルールを、会話や記憶に頼らず参照できる状態を保つ。

## 配置と新設ゲート

- **プロジェクトごとに vault の MOC フォルダへ1枚**: `<sub>-facts.md`（例: `AIads-facts.md`）。全プロジェクト共通の1枚は作らない（事実はプロジェクト固有）。
- **新設は最後の手段**: 作る前に既存台帳（`key_findings.md`・playbook 案件固有節・genba-rules 等）で吸収できないか必ず確認。既存正本があるプロジェクトに機械的に増設・改名しない。
- 新設したら MOC 索引 + 現場ルール導線（あれば）に1行登録。

## スコープ（冒頭に必ず明記・ここが境界の正本）

- **含む**: データ定義／正本の所在／換算係数（例: 管理画面CV=真CVの約3.1倍盛り）／損益・判定閾値／ユーザー裁定の運用ルール
- **含まない**: 分析知見（→ key_findings）／仮説・未検証（→ 答え合わせ後に昇格）／施策案（→ ボード/レポート）／媒体の一般定石（→ playbook）／決定の経緯（→ decisions）

**振り分けテスト（迷ったらこの3問）**:
1. 「分析の**結果**として何が分かったか」→ **finding**（key_findings へ）
2. 「分析・判断の**時にどの定義・数字・正本を使うか**」→ **F#（事実）**
3. 「そのうえで**何をしてよい/いけないか**」→ **R#（ルール）**

同じ内容を facts と key_findings の両方に書かない（必要なら相互リンクのみ）。

## エントリ規約（F と R でライフサイクルを分ける — 1枚統合・1ライフサイクル化はしない）

**共通**: 1エントリ=1行・確定日必須・同一対象の重複行禁止・長文化したら詳細を別ノートへ分離し台帳は要約+リンク。

| | **F#（事実）** | **R#（ユーザー裁定ルール）** |
|---|---|---|
| 成立条件 | 一次データ・実ファイルでの検証（根拠リンク必須） | ユーザーの裁定（裁定日・裁定の記録先を書く。根拠数値は任意） |
| 覆るとき | 反証データで**書換え**。旧値は行末 `（旧: …・supersededYYYY-MM-DD）` か「退役」節へ | ユーザーの再裁定でのみ変更・廃止 |
| mistakes 行き | **誤判断の原因と再発防止がある場合だけ**（事実の更新≠失敗） | 同左 |

## 追記の確定シグナル（AI の自動追記はこの3つのみ・推論や暫定値は追記禁止）

1. **ユーザーの明示裁定**（「〜はこうする」の一言）→ R# へ
2. **一次データでの答え合わせ完了**（実ファイル検証・実測）→ F# へ
3. **既存正本の更新の反映**（正本が変わった事実の同期）

追記したら**同じ応答で差分（追加した行）を報告**する。一時的指示（今回だけ）は追記しない — 迷ったら「今回だけ？今後も？」を確認。

## skeleton（新設時にコピー）

```markdown
# <sub> 自社ファクト＆ローカルルール台帳（1枚・living）

> [!todo] 🎯 この1枚の役割
> 「うちの数字はどれが本当か・うちだけのルールは何か」を数秒で引ける状態を保つ。
> 含む: データ定義・正本の所在・換算係数・閾値・ユーザー裁定ルール
> 含まない: 分析知見(→key_findings)・仮説・施策案・媒体定石(→playbook)・経緯(→decisions)

## §1 計測・データのファクト
| # | ファクト | 実務での使い分け | 根拠（正本） | 確定日 |
|--:|---|---|---|---|
| F1 | … | … | [[正本ノート]] / repo path | YYYY-MM-DD |

## §2 閾値・採算のローカルルール
## §3 運用のローカルルール（ユーザー裁定）
| # | ルール | 裁定の記録先 | 裁定日 |

> [!note] 追記のしかた（腐らせない3行）
> - 1エントリ=1行・確定日必須。F#は根拠リンク必須・R#はユーザー裁定のみで成立
> - 他に正本がある事実は要約+リンク（全文コピー禁止）
> - F#が覆ったら書換え+（旧:…・superseded日付）。mistakes 行きは誤判断があった時だけ
```

## 肥大防止（行数上限は設けない）

現役 current だけを本表に置く。superseded は「退役」節 or 年月別 archive へ。1行が長くなったら詳細分離。lint 等の機械ゲートは作らない（欠損が繰り返し実測されたら検討）。
