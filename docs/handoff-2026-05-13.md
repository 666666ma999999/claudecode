# 引き継ぎ書: 2026-05-13 セッション追加分 (2 台目 PC 用)

## 何を追加したか (3 件)

### 1. SessionStart version-diff hook
- ファイル: `~/.claude/hooks/sessionstart-version-diff.sh`
- 役割: `claude --version` 変化を検知し、コンテキスト注入で通知
- state: `~/.claude/state/claude-version.last`

### 2. Weekly env pulse (launchd 月曜 08:05)
- ファイル: `~/.claude/scripts/weekly_env_pulse.sh`
- plist: `~/Library/LaunchAgents/com.masa.weekly-env-pulse.plist` (git 管理外)
- 役割: 週次でローカル FS 観測 (新規 repo / アクティブ session / launchd 死活)
- 出力: `~/.claude/state/env_pulse.jsonl` (1 行 JSONL append、過去 4 週超は archive)

### 3. rules/10-git-and-execution-guard.md 追記
- 「新規禁止ルールの追加先優先順位」セクション (3 行)
- permissions.deny → hook → 二重防御 の判断順を明文化

## プロジェクト依存性 (重要)

- **依存なし**: hook/script はすべて汎用設計
  - version-diff: `claude --version` のみ参照
  - weekly_env_pulse: `~/Desktop` `~/Documents` を generic に find (プロジェクト固有パスなし)
- **2 台目で何もしなくて自動適応**: env_pulse.jsonl の出力内容は 2 台目のローカル repo に応じて変わる

## 2 台目 PC でのセットアップ

git sync で `~/.claude/` の差分を取り込んだ後、以下を 1 回実行:

```bash
bash ~/.claude/scripts/apply-session-additions-2026-05-13.sh
```

このスクリプトが行うこと:
1. plist を `~/Library/LaunchAgents/` にコピー (git 管理外のため)
2. `launchctl load` で登録
3. version-diff の state file を初期化 (`claude --version` の現在値を記録)
4. 動作確認 dry-run
