# コアワークフロー

## 事実確認ルール（最優先）

現状の事実に関する質問には**必ずツールで実態を確認してから回答**。
- 禁止: 推測・一般論での回答、ツール実行なしでの断定
- 手順: ツールで確認 → 確認結果に基づき回答 → 確認不可なら「未確認」と明示

## エラー報告フォーマット

1. 症状: エラーメッセージの正確な引用（Sentry URL該当時のみ）
2. 原因: コード追跡→根本原因（ファイル名:行番号）
3. 経緯: 過去の変更履歴・スキルとの不整合
4. 選択肢: 複数の修正方針（最低2つ）+ トレードオフ

## 実装完了チェック（自動発火）

Edit/Writeでコード変更後、ユーザーへの完了報告前に `implementation-checklist` スキルを実行。
除外: ドキュメントのみ、設定ファイルのみ、1行以下の軽微修正。

## タスク管理

task.mdベース。コマンド: `/launch-task`, `/onboard`, `/update-task`, `/review`, `/verify-step`。
詳細は `~/.claude/commands/` 参照。

## Docker-Only開発

依存管理・ビルド・実行はDocker経由。ホスト上 `pip install`, `npm install`, `npx` 等は禁止。
適用除外: MCP設定、Claude Codeツール拡張。

## メモリシステム

- Claude-Mem: 活動記録（自動）
- Memory MCP: 知識蓄積（意図的保存 — 「覚えておいて」、新方針決定時、再利用知見発見時）
