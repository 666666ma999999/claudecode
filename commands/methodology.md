# /methodology

現在のプロジェクトに「データ駆動で施策を回す」作業メソドロジーの雛形を置くコマンド（prime_suite で確立した型）。

## 実行

- 引数 `$ARGUMENTS` が配置先ディレクトリならそこ、空なら `docs/`（無ければ cwd）に置く:

```bash
bash ~/.claude/scripts/place-methodology.sh "$ARGUMENTS"
```

- 出力（配置先・次にやること）をそのまま見せる。既存ファイルがあれば上書きせずその旨を伝える。

## 配置後

- ユーザーが望めば、各ステップの「問い」（① 既存データは？ / ② 最新の実態は？ / ③ 公式確認したか？ / ④ 固定フォーマット・楽観補正・撤退基準は？ / ⑤ 反証・裏取りしたか？ / ⑥ 可視化したか？ / メタ層: 仕組みで守らせたか？）に、当プロジェクトのデータ・ツールで答える壁打ちを始める。
- 具体例の参照: Obsidian `[[prime_suite-methodology-draft]]`（prime_ad/crm の実タスク）。
