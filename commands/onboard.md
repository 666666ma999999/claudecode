# /onboard - セッション復帰コマンド

中断した作業を再開するためのコンテキスト復帰を行う。

## 実行手順

### 1. task.mdの確認

以下の場所でtask.mdを検索:
1. `.claude/workspace/task.md`（プロジェクト固有）
2. `task.md`（プロジェクトルート）

```bash
# 検索コマンド
find . -name "task.md" -maxdepth 3 2>/dev/null
```

### 2. task.mdが存在する場合

task.mdを読み込み、以下を要約して報告:

1. **タスク概要**: 目的・背景を1-2文で
2. **現在のステータス**: planning / in_progress / review / completed
3. **完了済み項目**: チェック済みの成功基準・実装計画
4. **前回の変更内容**: 「変更ログ」セクションから変更ファイル一覧を表示
5. **設計判断の確認**: 「意思決定記録」セクションから判断事項を提示（先祖返り防止）
6. **次のアクション**: 「次に実行すべき作業」セクションから
7. **未解決の問題**: 残っている課題

### 3. task.mdが存在しない場合

以下を提案:
- 「task.mdが見つかりません。新規タスクを開始しますか？」
- `/launch-task` コマンドの案内

### 4. 関連ファイルの確認

task.mdの「関連ファイル」セクションに記載されたファイルが存在するか確認。
存在しないファイルがあれば警告。

## 出力フォーマット

```
## セッション復帰完了

**タスク**: [タスク名]
**ステータス**: [ステータス]
**Issue**: [リンク]

### 前回までの進捗
- [完了項目1]
- [完了項目2]

### 次のアクション
1. [次のステップ1]
2. [次のステップ2]

### 未解決の問題
- [問題1]

---
作業を再開しますか？
```

## 注意事項

- task.mdは読み取り専用として扱う（この時点では更新しない）
- 関連ファイルの内容は必要に応じて読み込む（全て一度に読まない）
- ユーザーの確認後に作業を開始する
